# blog-post

SEO 최적화 블로그 포스팅을 멀티 에이전트 방식으로 생성한다.
티스토리 HTML + 마크다운 두 형식으로 저장한다.

## 실행 전 준비
blog-output/ 폴더가 없으면 프로젝트 루트에 생성한다.

## STEP 1 — [분석가 에이전트] 프로젝트 분석

아래 항목을 순서대로 직접 확인한다:

1. 프로젝트 구조 파악
   - 디렉토리 트리 확인 (depth 2~3)
   - package.json / pubspec.yaml 등 의존성 파일
   - README가 있으면 읽기

2. 핵심 구현 추출
   - 가장 복잡한 파일 / 로직 찾기
   - 커스텀 훅, 유틸, 핵심 컴포넌트 확인
   - 10~20줄 이내 인상적인 코드 스니펫 3개 이상 선별

3. 삽질 흔적 찾기 (이 부분이 글의 차별점)
   - 주석에 TODO, FIXME, 의문형 코멘트 등
   - 복잡하게 우회한 로직
   - 커밋 메시지 확인 가능하면 확인

4. 기술 스택 정리
   - 주요 라이브러리 버전 포함
   - 특이한 조합이나 선택 이유가 있으면 표시

분석 완료 후 아래 형식으로 요약 출력:

[분석가 리포트]
- 프로젝트 한 줄 설명:
- 핵심 기술 스택:
- 가장 인상적인 구현:
- 발견한 삽질 포인트:
- 선별한 코드 스니펫 (파일명 포함):
- 키워드 후보 (메인 1개 + 서브 3개):

## STEP 2 — [작가 에이전트] 초안 작성

분석가 리포트를 기반으로 초안을 작성한다.

글 구조 (반드시 이 순서):
- ## [도입] 어떤 문제를 해결하려 했나 (독자 공감 유도, 300자 이내)
- ## [프로젝트 개요] 뭘 만들었나, 왜 이 스택인가
- ## [핵심 구현 1] 가장 인상적이거나 어려운 부분 + 코드 스니펫
- ## [핵심 구현 2] 두 번째 포인트 + 코드 스니펫
- ## [삽질 & 해결] 실제로 막혔던 것 + 어떻게 해결했나
- ## [결과 & 회고] 잘된 것 / 아쉬운 것 / 다음엔 어떻게 할 것
- ## [마무리] 한 줄 요약 + 독자에게 남기는 말

작성 조건:
- 분량: 3,000자 이상
- 톤: 1인칭, 솔직한 개발자 후기 (AI 문체 금지)
- 코드 스니펫: 실제 프로젝트 파일에서 인용, 10~20줄 이내
- 잘 안 된 것도 솔직하게 서술
- 뻔한 서론 절대 금지 ("안녕하세요 오늘은 ~에 대해 알아보겠습니다")

SEO 조건:
- H1 제목에 메인 키워드 포함
- H2/H3 소제목 구조 명확히
- 이미지 삽입 위치를 [이미지: 설명] 형식으로 표시
- 내부 링크 추천 위치 표시 ([내부링크: 관련 포스팅 주제])

초안 완성 후 출력:
[작가 초안 완성]
제목 후보:
1.
2.
3.

## STEP 3 — [편집장 에이전트] 1차 검토

작가 초안을 검토하기 전, 먼저 아래 작업을 수행한다.

**[사전 작업] 핵심 주제 정의**
분석가 리포트의 "프로젝트 한 줄 설명"과 "가장 인상적인 구현"을 기반으로
이 글이 독자에게 전달해야 할 핵심 메시지 1문장을 먼저 정의한다.

예) "React에서 커스텀 훅으로 API 상태를 관리하다 마주친 무한 루프 문제와 해결 과정"

핵심 메시지를 정의한 뒤, 모든 검토 기준을 이 메시지 기준으로 판단한다.
핵심에서 벗어난 내용은 분량에 관계없이 필수 수정으로 지적한다.

칭찬은 생략하고 문제점과 개선 지시만 출력한다.

검토 기준:
- 각 섹션이 핵심 메시지와 직결되는가? (벗어난 섹션은 구체적으로 지목)
- 핵심 구현 설명이 피상적이지 않고 "왜 이렇게 했는가"까지 다루는가?
- 실제 코드가 인용됐는가? (임의 예시 코드면 지적)
- 삽질/실패 경험이 구체적으로 담겼는가? (막연한 "어려웠다"는 지적)
- 도입부가 핵심 문제를 즉시 제시하는가? (배경 설명만 늘어놓으면 지적)
- 다른 블로그에도 있는 일반론으로 채워진 섹션이 있는가?
- 키워드가 제목/첫 단락/소제목에 자연스럽게 포함됐는가?
- AI가 쓴 것 같은 문체가 있는가? (해당 문장 직접 인용해서 지적)
- 분량이 3,000자 이상인가?
- 이미지 삽입 위치가 표시됐는가?

## STEP 7 — 최종본 생성 및 저장

사용자 선택 반영 후 최종본을 저장한다.

파일명 규칙:
- 선택된 제목에서 핵심어 추출 → 영문 슬러그 변환
- 날짜 prefix 포함
- 예: 2026-03-24-claude-code-react-hooks-optimization

저장 경로:
blog-output/
├── 2026-03-24-{slug}.md
└── 2026-03-24-{slug}.html

마크다운 파일 상단 frontmatter:
---
title: (최종 제목)
date: (작성일)
keywords: (메인키워드, 서브키워드1, 서브키워드2, 서브키워드3)
description: (메타 디스크립션 160자 이내)
tags: (티스토리 태그 5개)
---

HTML 파일 조건:
- 티스토리 HTML 편집기에 바로 붙여넣기 가능한 형식
- h2, h3, p, pre, code, ul 태그 사용
- 인라인 스타일 없이 순수 HTML 태그만
- 코드 블록: <pre><code class="language-tsx"> 형식
